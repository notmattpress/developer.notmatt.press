widgets_init

do_action( ‘widgets_init’ )

Fires after all default WordPress widgets have been registered.

  1. Skip to note 3 content

    You should note that the widgets_init hook is fired as part of the init hook – with a priority of 1.

    This means that it will fire before any code you may add to the init hook with a default priority.

    • You will most probably run into this if you try to disable a widget registered by a plugin, if that plugin has added the widget using the widgets_init hook. You will need to add remove_widget($x,$y) through an init action with priority 0. Example add_action( 'init', 'wpdocs_my_init', 0 ); function wpdocs_my_init() { $terms = get_terms( 'ts_product_brand', array( 'hide_empty' => true ) ); if ( ! is_array( $terms ) || count( $terms ) < 2 ) { remove_action( 'widgets_init', 'ts_product_filter_by_brand_load_widget' ); } }

  2. Skip to note 4 content

    For example, used with registering sidebars:

    function mytheme_widgets_init() {
	register_sidebar( array(
		'name'          => __( 'Single Post Widgets', 'textdomain' ),
		'id'            => 'mytheme-single-post-widgets',
		'description'   => __( 'Widgets in this area will be shown under your single posts, before comments.', 'textdomain' ),
		'before_widget'	=> '',
		'after_widget'	=> '',
		'before_title'	=> '',
		'after_title'	=> '',
    ) );
}
add_action( 'widgets_init', 'mytheme_widgets_init' );
